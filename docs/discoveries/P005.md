## Key Findings

### `Commands::Dedup` clap variant (Anchor #6)

Confirmed at main.rs:31-38:
```rust
Dedup {
    #[arg(long)]
    state: PathBuf,
    #[arg(long = "rows-json")]
    rows_json: PathBuf,
},
```
Dispatch arm at main.rs:112 was `cli::dedup::run(state, rows_json)` — updated to full error-mapping arm in Task 3.

### `#![allow(dead_code)]` removal

Removed from state.rs:9 in Task 1. After removal: `StateFile` + `SCHEMA_VERSION` consumed by new `pub fn read()` + `cli::dedup::run()`. `cargo clippy --all-targets -- -D warnings` passes clean — no `dead_code` warning emitted. Confirms wire-in complete.

### `tempfile = "3"` in `[dependencies]` (Anchor #9)

Found at Cargo.toml:21 in the `[dependencies]` section (not `[dev-dependencies]`). Available for both unit tests and integration tests without additional config. `tempfile::NamedTempFile::new()` used directly in state.rs `#[cfg(test)]` module and `tests/dedup_cli.rs`.

### `serde_json::json!` key ordering (P004 Discovery carried forward)

`serde_json::json!` macro alphabetises output keys. Actual stdout key order: `"kept"` → `"observed_ids"` → `"skipped"`. Integration tests use `predicate::str::contains` substring match, NOT exact JSON equality — consistent with P004 pattern.

### `anyhow::Error::is::<StateReadError>()` downcast

`StateReadError` derives `thiserror::Error` + `Debug` and implements `std::error::Error + Send + Sync + 'static`. When `?` propagates `StateReadError` through `anyhow::Result`, `e.is::<StateReadError>()` correctly identifies it. Verified: state file missing → exit 1, rows malformed → exit 2.

### README update (Anchor #14 conditional)

`dedup` was present in README at line 37, but only as a stub line under "Other subcommands (logic wired in P005-P011)". Per phiếu Task 6.3 rule: "subcmd activated → treated as subcmd added → Tầng 1 doc update mandatory." Worker added full quick-start section with exit code table matching ARCHITECTURE §1. Old stub line removed.

### `run()` function parameter naming

Phiếu Task 2 spec: `state_path: PathBuf` (renamed from `state`) to avoid shadowing the `state` module import. Implemented exactly as specified. Main.rs dispatch passes the clap `state` field as first positional argument to `run(state, rows_json)` — clap variant field `state: PathBuf` maps cleanly.

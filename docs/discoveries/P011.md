## Phát hiện quan trọng

### [mechanical] O1.1 — Import path `Parameters` + `Json` sai trong phiếu draft

**Phiếu spec:** `rmcp::handler::server::router::tool::{Json, Parameters}`

**Thực tế từ rmcp-1.7.0 source:**
- `rmcp/src/handler/server/router/tool.rs` doc comment line 9: `handler::server::{wrapper::{Parameters, Json}, tool::ToolRouter}`
- `Parameters` = `handler/server/wrapper/parameters.rs`
- `Json` = `handler/server/wrapper/json.rs`, also re-exported as `rmcp::Json` top-level (lib.rs:18)

**Fix (Tầng 2):** Dùng đúng path:
```rust
use rmcp::{Json, handler::server::wrapper::Parameters};
```

### [mechanical] O1.2 — `#[tool_router(server_handler)]` tạo `from_build_env()` đọc tên crate rmcp, không phải advisory-inbox

**Phiếu spec:** Macro auto-generates `get_info()` đọc từ `Cargo.toml` của binary.

**Thực tế từ rmcp-macros-1.7.0:**
- Macro generated code gọi `rmcp::model::Implementation::from_build_env()`
- `from_build_env()` trong rmcp source dùng `env!("CARGO_CRATE_NAME")` — nhưng env var này expand ở COMPILE TIME trong context của rmcp crate → trả về "rmcp", không phải "advisory-inbox"
- Integration test `serve_responds_to_initialize_with_server_info` FAIL với `"name":"rmcp"` trước fix

**Fix (Tầng 2):** Dùng two-step pattern:
```rust
#[tool_router]
impl AdvisoryInboxService { /* 6 tools */ }

#[tool_handler]
impl ServerHandler for AdvisoryInboxService {
    fn get_info(&self) -> ServerInfo {
        ServerInfo::new(ServerCapabilities::builder().enable_tools().build())
            .with_server_info(Implementation::new(
                env!("CARGO_PKG_NAME"),    // "advisory-inbox" — expands in OUR crate
                env!("CARGO_PKG_VERSION"),
            ))
    }
}
```

## Sai lệch so với phiếu

### [Tầng 2] dead_code lint — scaffold types chưa có consumer

**Phiếu giả định:** "cargo build --release — zero warnings" có thể đạt mà không cần attribute bổ sung.

**Thực tế:** 5 rustc `dead_code` warnings xuất hiện khi build `--release` và clippy `-D warnings`:
- `Status`, `Severity`, `AdvisoryRow` trong `row.rs` — không ai construct ngoài `#[cfg(test)]`
- `SCHEMA_VERSION`, `StateFile` trong `state.rs` — tương tự

**Nguyên nhân:** Các type `pub` trong internal module (`mod row; mod state;`) nhưng binary code path chưa dùng (toàn bộ `cli/*.rs` vẫn là stub printf). Rust lint `dead_code` áp dụng ở binary crate level.

**Quyết định Tầng 2 (Worker tự quyết):** Thêm `#![allow(dead_code)]` ở module level (crate-inner attribute) với comment giải thích scaffold intent. Đây là rustc lint không phải clippy lint; phiếu constraint #8 cấm clippy escape hatches, không cấm rustc lint attributes cho scaffold code. Attribute sẽ được xóa khi P004+ wire-in import type này.

**Không escalate** vì: không đổi type signature, không đổi CLI surface, không đổi schema — chỉ là lint suppress cho scaffold phase.
